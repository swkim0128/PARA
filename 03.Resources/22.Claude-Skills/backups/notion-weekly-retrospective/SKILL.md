# 노션 주간 회고 작성 스킬

## 개요
'일지 및 회고' 데이터베이스에서 주간 일정 페이지를 찾아 해당 주에 기록된 WORK/LIFE 활동을 분석하고, KPT(Keep, Problem, Try) 템플릿으로 회고록을 작성하여 페이지 하단에 추가하는 스킬입니다.

## 사용 시나리오
```
사용자: "이번 주 회고 작성해줘"
→ 이번 주 일정 페이지 찾기
→ WORK/LIFE 섹션 내용 분석
→ KPT 템플릿으로 회고 작성
→ 페이지 하단에 회고 추가
```

```json
{
```

- 미완료 항목 `[ ]` 개수
- 카테고리별 진행률 파악

## 4단계: KPT 회고 템플릿

### KPT 프레임워크 설명
- **Keep (유지)**: 잘 했고, 앞으로도 계속 유지할 것들
- **Problem (문제)**: 문제가 되었거나 개선이 필요한 것들
- **Try (시도)**: 다음에 시도해볼 것들, 개선 방안

### 회고 템플릿 구조
```markdown
## 📝 주간 회고

### Keep (유지할 것) ✅
- **[주제]**: [구체적인 내용]
- **[주제]**: [구체적인 내용]

### Problem (개선할 것) ⚠️
- **[주제]**: [구체적인 내용]
- **[주제]**: [구체적인 내용]

### Try (시도할 것) 🎯
- **[주제]**: [구체적인 내용]
- **[주제]**: [구체적인 내용]

### 한 줄 소감
> [이번 주를 한 마디로 정리]
```

## 5단계: 회고 작성 워크플로우

### 전체 프로세스
1. **주간 페이지 찾기**: Search로 해당 주차 페이지 검색
2. **페이지 내용 조회**: Fetch로 페이지 전체 내용 확인
3. **일정 내용 분석**: WORK/LIFE 체크박스 및 Week Things 분석
4. **KPT 회고 작성**: 분석 내용 기반 회고 생성
5. **회고 추가**: 페이지 하단 `<empty-block/>` 이후에 추가
6. **결과 확인**: 추가된 내용 확인

### 회고 분석 가이드라인

#### Keep 작성 기준
- `[x]` 완료된 업무/일정 중 의미있는 것
- 꾸준히 유지한 습관 (스터디, 운동 등)
- 효율적으로 처리한 업무
- 워라밸 유지 활동

#### Problem 작성 기준
- `[ ]` 미완료된 계획이나 일정
- 동시 진행으로 인한 집중도 저하
- 예상보다 오래 걸린 업무
- 놓친 개인 활동 (운동 등)

#### Try 작성 기준
- Problem에서 도출된 개선 방안
- 다음 주 우선순위 조정
- 미완료 업무 완료 계획
- 습관 형성/개선 계획

**4단계: KPT 회고 작성**

```markdown
## 📝 주간 회고

### Keep (유지할 것) ✅
- **업무 이슈 대응력**: 연동몰 가격입력 오류를 빠르게 파악하고 원인을 확인함
- **스크립트 자동화**: 통합몰 미등록 업체 추출 스크립트를 작성하여 반복 작업 효율화
- **운영 배포 완료**: 계획한 운영서버 배포를 무사히 마침
- **사이드 프로젝트 시작**: Grafana 기본 학습을 시작하며 모니터링 역량 강화 준비
- **워라밸 유지**: 스터디 모임, 집회 참여, 영화 관람 등 개인 시간 확보

### Problem (개선할 것) ⚠️
- **동시 진행 업무 과다**: 협력사, Billing, OJT, 플랫폼 전환 등 여러 업무가 병렬로 진행되어 집중도 저하
- **운동 루틴 실패**: 계획한 클라이밍을 실행하지 못함
- **MPADMIN 개선 지연**: 수요일~목요일 작업했으나 완료하지 못함
- **OJT 준비 미진행**: 신규 입사자 OJT 설정이 전혀 진행되지 않음

### Try (시도할 것) 🎯
- **업무 우선순위 명확화**: 다음 주 초에 업무별 우선순위를 정하고, 하나씩 완료하기
- **운동 일정 고정**: 클라이밍 요일을 확정하고 캘린더에 블로킹
- **OJT 집중 처리**: 네이버웍스/계정/개발환경 설정을 월~화 내 완료 목표
- **Grafana 학습 이어가기**: 데이터 소스 연결 및 대시보드 패널 구성 실습

### 한 줄 소감
> 여러 업무를 병행하느라 정신없었지만, 주말에 집회와 영화로 리프레시했다. 다음 주는 우선순위를 명확히 해서 하나씩 완료해 나가자.
```

## 9단계: 회고 품질 가이드라인

### 좋은 회고의 특징

**1. 구체적인 내용**
```
❌ "열심히 일했다"
✅ "연동몰 가격입력 오류를 빠르게 파악하고 원인을 확인함"
```

**2. 실행 가능한 Try**
```
❌ "더 잘하자"
✅ "네이버웍스/계정/개발환경 설정을 월~화 내 완료 목표"
```

**3. 균형 잡힌 관점**
- WORK와 LIFE 모두 다루기
- 긍정적인 점과 개선점 균형있게
- 과거 회고의 Try가 실행되었는지 확인

**4. 적절한 분량**
- Keep: 4-6개 항목
- Problem: 3-5개 항목
- Try: 3-5개 항목
- 한 줄 소감: 1-2문장

### 분석 시 고려사항

**체크박스 분석**:
- `[x]` 완료 → Keep 후보
- `[ ]` 미완료 → Problem/Try 후보
- 하위 항목 진행률 고려

**Week Things 분석**:
- 실제 요일별 수행 활동 확인

- 집중 업무 시간대 파악

### 2. 사용자 의도 파악
- "자세히" 요청 → 더 구체적인 회고
- "간단히" 요청 → 핵심만 담은 회고
- 특정 주제 강조 요청 시 반영

### 3. 일관된 톤 유지
- 긍정적이면서 객관적인 톤
- 자기비판보다 성장 관점
- 다음 주에 대한 희망적 마무리

### 4. 에러 방지
- 검색 시 "일지" vs "식단" 구분
- 페이지 존재 여부 먼저 확인
- 회고 중복 여부 확인
- selection 문자열 정확히 매칭

## 에러 처리

### 일반적인 에러와 해결방법

**에러**: "해당 주차 페이지를 찾을 수 없습니다"
- **원인**: 해당 주차의 일정 페이지가 없음
- **해결**: 사용자에게 알리고, 페이지 생성 여부 확인

**에러**: "페이지에 기록된 일정이 없습니다"
- **원인**: Week List가 비어있음
- **해결**: 사용자에게 간단한 회고라도 작성할지 확인

**에러**: "회고 추가 위치를 찾을 수 없습니다"
- **원인**: 페이지 구조가 예상과 다름 (`<empty-block/>` 없음)
- **해결**: Copyright 직전이나 페이지 최하단에 추가

**에러**: "이미 회고가 존재합니다"
- **원인**: 동일 주차에 회고가 이미 작성됨
- **해결**: 사용자에게 덮어쓰기/추가/유지 선택 요청

## 체크리스트

회고 작성 시 확인사항:
- [ ] 해당 주차 페이지가 존재하는지 확인 (일지 vs 식단 구분)
- [ ] 페이지 전체 내용을 fetch로 조회
- [ ] Week List 체크박스 상태 분석 (완료/미완료)

- [ ] 기존 회고 존재 여부 확인
- [ ] KPT 각 항목 작성 (Keep 4-6, Problem 3-5, Try 3-5)
- [ ] 한 줄 소감 포함
- [ ] `<empty-block/>` 이후에 회고 추가
- [ ] 추가 후 결과 확인
