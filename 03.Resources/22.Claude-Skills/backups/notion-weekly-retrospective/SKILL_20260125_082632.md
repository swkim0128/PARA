# 노션 주간 회고 작성 스킬

## 개요
'일지 및 회고' 데이터베이스에서 주간 일정 페이지를 찾아 해당 주에 기록된 WORK/LIFE 활동을 분석하고, KPT(Keep, Problem, Try) 템플릿으로 회고록을 작성하여 페이지 하단에 추가하는 스킬입니다.

## 사용 시나리오
```
사용자: "이번 주 회고 작성해줘"
→ 이번 주 일정 페이지 찾기
→ WORK/LIFE 섹션 내용 분석
→ KPT 템플릿으로 회고 작성
→ 페이지 하단에 회고 추가
```

## 2단계: 해당 주 일정 페이지 찾기

### 방법 1: 현재 주차 기준 검색 (기본)
현재 날짜를 기준으로 이번 주의 주차를 계산하여 검색합니다.

```
1. 오늘 날짜 확인 (예: 2025-01-25)
2. 이번 주 주차 계산 (예: week 4)
3. 검색 쿼리 생성 (예: "week 4")
```

**Notion Search 도구 사용**:
```json
{
  "query": "week 4",
  "query_type": "internal"
}
```

### 방법 2: 특정 주차 지정 검색
사용자가 특정 주차를 지정한 경우:

```
사용자: "지난 주 회고 작성해줘"
→ 현재 주차 - 1로 계산
→ 해당 주차 검색
```

```
사용자: "3주 전 회고 작성해줘"
→ 현재 주차 - 3으로 계산
→ 해당 주차 검색
```

**중요**: 검색 결과에서 페이지 URL/ID를 확인하고, 해당 페이지를 다시 fetch하여 전체 내용을 확인해야 합니다.

   - 운동/건강 관리

   - 사회적 활동 (약속 등)

3. **Weekly Summary 분석**
   - 이미 작성된 요약이 있는지 확인
   - 주요 하이라이트 파악

## 4단계: KPT 회고 템플릿

### KPT 프레임워크 설명
- **Keep (유지)**: 잘 했고, 앞으로도 계속 유지할 것들
- **Problem (문제)**: 문제가 되었거나 개선이 필요한 것들
- **Try (시도)**: 다음에 시도해볼 것들, 개선 방안

### 회고 템플릿 구조
```markdown
---

## 📝 주간 회고

### Keep (유지할 것) ✅
- [잘 한 점 1]
- [잘 한 점 2]
- [계속 유지할 습관/활동]

### Problem (개선할 것) ⚠️
- [문제점 1]
- [개선 필요 사항]
- [아쉬웠던 점]

### Try (시도할 것) 🎯
- [다음 주에 시도해볼 것]
- [개선 방안]
- [새로운 목표]

### 한 줄 소감
> [이번 주를 한 마디로 정리]
```

## 5단계: 회고 작성 워크플로우

### 전체 프로세스
1. **주간 페이지 찾기**: Search로 해당 주차 페이지 검색
2. **페이지 내용 조회**: Fetch로 페이지 전체 내용 확인
3. **일정 내용 분석**: WORK/LIFE 섹션별 활동 분석
4. **KPT 회고 작성**: 분석 내용 기반 회고 생성
5. **회고 추가**: 페이지 하단에 회고 섹션 추가
6. **결과 확인**: 추가된 내용 확인

### 회고 분석 가이드라인

#### Keep 작성 기준
- 계획대로 완료한 업무/일정
- 꾸준히 유지한 습관 (운동, 학습 등)
- 긍정적 피드백을 받은 활동
- 생산성이 높았던 순간

#### Problem 작성 기준
- 미완료된 계획이나 일정
- 시간 관리가 어려웠던 부분
- 예상과 다르게 진행된 업무
- 에너지 소모가 컸던 활동

#### Try 작성 기준
- Problem에서 도출된 개선 방안
- 새롭게 도전해볼 활동
- 다음 주 우선순위 조정
- 습관 형성/개선 계획

## 6단계: 회고 추가 실행

### 회고 삽입 위치
회고는 페이지 최하단에 추가합니다.

**위치 찾기 우선순위**:
1. `### MEMO` 섹션 이후 (Weekly Summary > MEMO 다음)
2. `## Weekly Summary` 섹션 이후
3. 마지막 요일 섹션 이후 (## 📅 일요일... 다음)

### 회고 추가 예제

**페이지 ID 확인 후 추가**:
```json
{
  "page_id": "찾은-페이지-id",
  "command": "insert_content_after",
  "selection_with_ellipsis": "### MEMO\n...",
  "new_str": "\n\n---\n\n## 📝 주간 회고\n\n### Keep (유지할 것) ✅\n- 매일 아침 운동 루틴 유지\n- 프로젝트 마감일 준수\n- 팀 미팅에서 적극적 의견 제시\n\n### Problem (개선할 것) ⚠️\n- 야근으로 인한 피로 누적\n- 개인 학습 시간 부족\n- 회의 시간이 길어지는 경향\n\n### Try (시도할 것) 🎯\n- 업무 시간 내 집중력 높이기\n- 점심시간 활용한 짧은 학습\n- 회의 전 안건 미리 정리하기\n\n### 한 줄 소감\n> 바쁘지만 의미 있는 한 주였다. 다음 주는 조금 더 여유를 갖자."
}
```

**5단계: KPT 회고 작성**

```markdown

## 📝 주간 회고

### Keep (유지할 것) ✅
- 월요일 아침 운동으로 한 주 시작하기
- 프로젝트 킥오프와 배포까지 계획대로 진행
- 기술 스터디 꾸준히 참여
- 목요일 저녁 독서 시간 확보

### Problem (개선할 것) ⚠️
- 수요일 문서 작성에 예상보다 많은 시간 소요
- 주 중반 운동 루틴 유지 실패
- 클라이언트 미팅 준비 시간 부족했음

### Try (시도할 것) 🎯
- 문서 템플릿 미리 준비해두기
- 화요일/목요일도 짧게라도 운동하기
- 미팅 전날 안건 및 자료 미리 체크하기

### 한 줄 소감
> 프로젝트 마무리까지 열심히 달렸던 한 주. 다음 주는 체력 관리에 더 신경쓰자.
```

## 9단계: 회고 품질 가이드라인

### 좋은 회고의 특징

**1. 구체적인 내용**
```
❌ "열심히 일했다"
✅ "3개의 기능 개발을 완료하고 코드 리뷰까지 마쳤다"
```

**2. 실행 가능한 Try**
```
❌ "더 잘하자"
✅ "매일 오전 30분은 집중 업무 시간으로 확보하기"
```

**3. 균형 잡힌 관점**
- WORK와 LIFE 모두 다루기
- 긍정적인 점과 개선점 균형있게
- 과거 회고의 Try가 실행되었는지 확인

**4. 적절한 분량**
- Keep: 3-5개 항목
- Problem: 2-4개 항목
- Try: 2-4개 항목
- 한 줄 소감: 1-2문장

### 분석 시 고려사항

**일정 빈도 체크**:
- 매일 반복된 활동 → Keep 후보
- 계획되었으나 누락된 활동 → Problem 후보
- 새롭게 시도한 활동 → 성공/실패에 따라 분류

**패턴 인식**:
- 주 초반 vs 후반 생산성 비교
- 업무 집중도 높은 요일/시간대
- 휴식/재충전 활동 분포

## 에러 처리

### 일반적인 에러와 해결방법

**에러**: "해당 주차 페이지를 찾을 수 없습니다"
- **원인**: 해당 주차의 일정 페이지가 없음
- **해결**: 사용자에게 알리고, 페이지 생성 여부 확인

**에러**: "페이지에 기록된 일정이 없습니다"
- **원인**: 일정이 비어있음
- **해결**: 사용자에게 간단한 회고라도 작성할지 확인

**에러**: "회고 추가 위치를 찾을 수 없습니다"
- **원인**: 페이지 구조가 예상과 다름
- **해결**: 페이지 최하단에 추가하도록 대체

**에러**: "이미 회고가 존재합니다"
- **원인**: 동일 주차에 회고가 이미 작성됨
- **해결**: 사용자에게 덮어쓰기/추가/유지 선택 요청

---

## 체크리스트

회고 작성 시 확인사항:
- [ ] 해당 주차 페이지가 존재하는지 확인
- [ ] 페이지 전체 내용을 fetch로 조회
- [ ] WORK/LIFE 섹션 내용 충분히 분석
- [ ] 기존 회고 존재 여부 확인
- [ ] KPT 각 항목 3개 이상 작성
- [ ] 한 줄 소감 포함
- [ ] 적절한 위치에 회고 추가
- [ ] 추가 후 결과 확인

---

## 마치며

이 스킬을 사용하면:
✅ 주간 일정을 기반으로 체계적인 회고 작성
✅ KPT 템플릿으로 성장 지향적 회고
✅ 노션 페이지에 자동으로 회고 추가
✅ 지난 주차 회고도 소급 작성 가능

**시작하기**:
- "이번 주 회고 작성해줘"
- "지난 주 회고 작성해줘"
- "week 3 회고 작성해줘"
- "이번 주 회고 작성해줘. 특히 프로젝트 완료 관련해서"
