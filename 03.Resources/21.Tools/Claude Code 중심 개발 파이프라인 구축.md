도구 파편화를 줄이고 하나의 AI 에이전트가 이슈 생성부터 테스트, 배포까지 전체 컨텍스트를 유지하게 만들면 워크플로우가 훨씬 더 강력하고 매끄러워질 것입니다. Claude Code는 내부적으로 터미널 명령어를 실행하고 테스트 결과를 읽어올 수 있기 때문에 단독으로도 이 모든 과정을 충분히 소화할 수 있습니다.

Copilot CLI 내용을 완전히 걷어내고, 오직 Claude Code 중심의 아키텍처로 수정한 **\[1. 개요 및 파이프라인 아키텍처\]** 내용을 전달해 드립니다.

**1\. 개요 및 파이프라인 아키텍처 🏗️**

## **1.1 문서의 목적**

본 매뉴얼은 우리 개발 팀이 **Claude Code 단일 도구**를 활용하여 기획(이슈 생성)부터 개발, 테스트, 배포, 그리고 사후 문서 동기화까지의 12단계 전 과정을 자동화하는 파이프라인을 구축하고 운용하는 방법을 안내합니다. 하나의 강력한 AI 에이전트를 통해 컨텍스트 단절 없이 개발 생산성을 극대화하는 것을 목표로 합니다.

## **1.2 기대 효과**

* **단일 툴체인(Toolchain) 구축:** 여러 AI 도구를 오갈 필요 없이, 터미널 내에서 Claude Code 명령어 하나만으로 코드 작성과 시스템 제어를 모두 수행할 수 있습니다.  
* **완벽한 컨텍스트 유지:** 이슈의 요구사항, 작성된 소스 코드, 터미널의 테스트 결과가 Claude Code의 단일 세션에 유지되어 훨씬 정확한 오류 수정과 문서화가 가능합니다.  
* **표준화된 워크플로우:** 팀원 누구나 공통된 환경(MCP)과 스킬(skills)을 사용하여 일관된 고품질의 개발 프로세스를 따를 수 있습니다.

## **1.3 파이프라인 아키텍처 및 핵심 요소**

우리의 자동화 파이프라인은 12단계의 개발 주기를 4개의 페이즈(Phase)로 나누어 처리하며, 오직 **Claude Code**가 다음 요소들을 활용해 전체 워크플로우를 주도합니다.

1. **Claude Code (핵심 AI 에이전트):**  
   * **역할:** 전체 워크플로우의 두뇌이자 실행기입니다. 자연어 명령을 이해하고 스스로 터미널 명령어를 실행하며, 파일 시스템에 접근합니다.  
   * **주요 담당 파이프라인:** \- 작업이슈 생성 및 문서화  
     * 소스 코드 분석 및 직접 수정  
     * **기능 테스트 스크립트 자체 실행 및 결과 분석**  
     * Git 커밋/푸시 및 CI/CD(배포) 트리거  
     * 사후 문서 동기화  
2. **MCP 서버 (Model Context Protocol):**  
   * **역할:** Claude Code가 이슈 트래커(GitHub/Jira) 등 외부 서비스와 안전하게 직접 통신할 수 있도록 연결해 주는 표준 플러그인 환경입니다.  
3. **오픈 에이전트 스킬 (Skills):**  
   * **역할:** 팀의 고유한 개발 표준, 테스트 실행 방법론, 파이프라인 순서 등을 Claude Code가 언제든 꺼내 쓸 수 있도록 정의해 둔 지침서 및 자동화 템플릿입니다.
