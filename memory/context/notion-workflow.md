# Notion Workflow

seongwoo의 노션 기반 관리 체계 정리.

## 역할 분담: Obsidian vs Notion

| 도구 | 역할 |
|------|------|
| **Obsidian** | 학습 노트, 기술 리소스, 사이드 프로젝트 문서 (PARA 방법론) |
| **Notion** | 일정 관리, 프로젝트/업무 추적, 식단 관리 |

→ Obsidian은 **지식 저장소**, Notion은 **실행/관리 도구**로 분리 운영.

## Notion 데이터베이스 맵

### 1. 일정 관리
- **Diary** (일지 및 회고)
  - 주간 일정 기록
  - KPT 회고록 작성 (Keep, Problem, Try)
  - 관련 스킬: `notion-weekly-schedule`, `notion-weekly-retrospective`

### 2. 프로젝트 관리
- **Projects** — 프로젝트 목록 및 상태
- **Tasks** — 세부 업무
- **Sprints** — 스프린트 단위 관리
- 관련 스킬: `notion-project-creator`

### 3. 식단 관리 (Areas)
- **Diet** — 식단 기록
- **Ingredients** — 식재료
- **Foods** — 음식 목록

## 사용 가능한 자동화 스킬
| 스킬 | 기능 |
|------|------|
| `notion-weekly-schedule` | 이번 주 Diary 페이지에 일정 추가 |
| `notion-weekly-retrospective` | 주간 KPT 회고록 자동 작성 |
| `notion-project-creator` | Projects DB에 프로젝트 생성 + Tasks 추가 |
