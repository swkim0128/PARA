## 예상 이슈 및 고려사항

### 1. API Rate Limiting
**문제**: 동행복권 API 호출 제한
**대응**: 
- 요청 간 딜레이 추가 (time.sleep)
- 실패 시 재시도 로직 (exponential backoff)
- 로컬 캐싱으로 중복 요청 방지

### 2. 데이터 무결성
**문제**: 크롤링 중단 시 불완전한 데이터
**대응**:
- 트랜잭션 사용
- 데이터 검증 로직 추가
- 중복 체크 (UNIQUE 제약조건)

### 3. 스케줄러 안정성
**문제**: 장기 실행 시 메모리 누수 또는 크래시
**대응**:
- 로깅 시스템 구축
- 예외 처리 강화
- 헬스체크 메커니즘

### 4. 추천 알고리즘 정확도
**문제**: 통계적 의미 없는 추천
**대응**:
- 다양한 가중치 조합 실험
- A/B 테스트로 검증
- 사용자 피드백 수집
