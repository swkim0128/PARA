## 1. 계층적 권한 관리: RBAC와 팀 동기화

Grafana의 거버넌스는 사용자의 역할에 따라 리소스 접근 범위를 엄격히 제한하는 **역할 기반 액세스 제어(RBAC)**에서 시작됩니다.

- **표준화된 역할:** 기본적으로 Viewer, Editor, Admin 역할을 통해 대시보드 조회, 수정, 설정 변경 권한을 제어합니다.

- **엔터프라이즈 세분화:** 엔터프라이즈 환경에서는 단순한 역할을 넘어, 특정 대시보드 폴더나 데이터 소스에 대해 정교한 커스텀 권한을 설정할 수 있습니다. 이를 통해 민감한 재무 데이터나 보안 로그가 포함된 대시보드는 특정 관리자 그룹만 접근하도록 격리할 수 있습니다.

- **팀 동기화 (Team Sync):** LDAP, SAML, OAuth2와 같은 외부 인증 공급자(IDP)와 연동하여, 사내 조직 구조(Active Directory 그룹 등)를 Grafana 팀 구성과 자동으로 동기화합니다. 이는 입/퇴사자에 대한 접근 권한 관리를 자동화하여 보안 구멍을 방지합니다.

## 2. 데이터 수준의 격리: LBAC (Label-Based Access Control)

단순히 대시보드 접근을 막는 것만으로는 부족할 때, Grafana는 **데이터 소스 내부의 콘텐츠**를 제어하는 **레이블 기반 액세스 제어(LBAC)**를 제공합니다.

- **데이터 거버넌스:** LBAC를 사용하면 동일한 Loki 로그 데이터 소스를 사용하더라도, 'Team A' 사용자는 `env=prod, team=A` 레이블이 붙은 로그만 볼 수 있고, 'Team B'의 데이터는 쿼리조차 할 수 없게 설정할 수 있습니다.

- **개인정보 보호 및 컴플라이언스:** 이는 GDPR이나 HIPAA와 같은 규제 준수가 필요한 환경에서, 민감한 정보가 포함된 로그나 메트릭이 인가되지 않은 사용자에게 노출되는 것을 원천적으로 차단하는 강력한 수단입니다.

## 3. 기계 및 자동화 계정 관리: 서비스 계정 (Service Accounts)

DevOps 및 GitOps 환경에서는 사람이 아닌 시스템(CI/CD 파이프라인, Terraform 등)이 Grafana에 접근해야 합니다.

- **API 키의 대체:** 기존의 API 키 방식을 대체하여 **서비스 계정**을 사용합니다. 이는 실제 사용자와 분리된 기계 전용 계정으로, 특정 권한과 범위를 가진 토큰을 발급하여 자동화 스크립트가 과도한 권한을 갖지 않도록 제어합니다.

- **자동화된 프로비저닝:** Terraform을 사용하여 이러한 서비스 계정과 권한 설정 자체를 코드로 관리(IaC)함으로써, 권한 부여 과정의 투명성을 확보하고 무단 변경을 추적할 수 있습니다.

## 4. 인프라 보안 및 비밀 관리 (Secrets Management)

액세스 제어는 사용자뿐만 아니라 데이터 소스 연결 정보에 대한 보안도 포함합니다.

- **비밀 정보 암호화:** 데이터베이스 비밀번호나 API 토큰과 같은 민감한 정보는 Grafana 내부에 평문으로 저장되지 않습니다. 엔터프라이즈 환경에서는 AWS KMS, Azure Key Vault, HashiCorp Vault와 같은 외부 키 관리 시스템(KMS)과 통합하여 데이터 소스 연결 정보를 암호화하고 관리합니다.

- **데이터 소스 보안:** MySQL과 같은 데이터 소스 연결 시, `DROP`이나 `DELETE` 권한이 없는 읽기 전용 계정을 사용하여 만약의 사태에 Grafana가 공격 통로로 악용되는 것을 방지합니다.

## 5. 감사 및 모니터링 (Auditing)

거버넌스의 완성은 **"누가 언제 무엇을 했는가"**를 추적하는 것입니다.

- **감사 로그 (Audit Logs):** 사용자가 대시보드를 조회하거나, 설정을 변경하거나, 권한을 수정하는 모든 활동을 기록합니다. 이는 보안 사고 발생 시 포렌식 분석을 가능하게 하고, 내부 통제 정책 준수 여부를 증명하는 데 사용됩니다.

## 요약

보안 및 거버넌스 맥락에서 **액세스 제어**는 단순한 로그인 기능이 아닙니다. 이는 **1) RBAC/LBAC를 통한 사용자 및 데이터 격리, 2) 외부 IDP와의 통합을 통한 중앙 집중식 관리, 3) KMS를 활용한 인프라 자격 증명 보호, 4) 서비스 계정을 통한 안전한 자동화**를 포괄하는 **종합적인 보안 아키텍처**입니다.
