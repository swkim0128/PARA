제공된 소스들을 바탕으로 쿠버네티스 환경에서 **Prometheus와 Grafana를 설치**하고, **데이터 소스를 연결**하는 전체 과정을 정리해 드립니다.  
가장 표준적이고 운영 효율적인 방식인 \*\*Helm 차트(kube-prometheus-stack)\*\*를 사용하는 절차를 기준으로, 실제 프로덕션 환경에서 고려해야 할 설정 파일(values.yaml) 예시와 명령어를 포함하여 상세히 설명합니다.

### 1\. 사전 준비 (Prerequisites)

설치를 시작하기 전에 다음 도구들이 준비되어 있어야 합니다.

* **Kubernetes Cluster**: 실행 중인 클러스터  
* **kubectl**: 클러스터와 통신하기 위한 CLI 도구  
* **Helm**: 쿠버네티스 패키지 매니저 (v3 이상)

### 2\. Prometheus & Grafana 통합 설치 (Helm 활용)

단일 컨테이너로 각각 설치하는 것보다 kube-prometheus-stack을 사용하면 **Prometheus, Grafana, Alertmanager, Node Exporter, Prometheus Operator**를 한 번에 통합 설치할 수 있어 운영 관리에 매우 효율적입니다 1\.

#### 2.1 Helm 리포지토리 추가 및 업데이트

Prometheus 커뮤니티의 공식 차트 저장소를 추가합니다.  
\# 리포지토리 추가  
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts

\# 리포지토리 최신화  
helm repo update

#### 2.2 네임스페이스 생성

모니터링 리소스를 격리하여 관리하기 위해 monitoring 네임스페이스를 생성합니다 3\.  
kubectl create namespace monitoring

#### 2.3 커스텀 설정 파일 작성 (선택 사항이지만 권장)

기본 설정으로 설치해도 되지만, \*\*데이터 영속성(Persistence)\*\*과 **관리자 비밀번호** 설정을 위해 values.yaml 파일을 작성하는 것이 좋습니다 5\.  
my-values.yaml 파일을 생성하고 아래 내용을 저장합니다.  
\# my-values.yaml 예시  
prometheus:  
  prometheusSpec:  
    storageSpec: \# 데이터 영속성 설정 (재시작 시 데이터 유지)  
      volumeClaimTemplate:  
        spec:  
          storageClassName: standard \# 클러스터의 스토리지 클래스명 확인 필요  
          accessModes: \["ReadWriteOnce"\]  
          resources:  
            requests:  
              storage: 10Gi  
    retention: 10d \# 데이터 보관 기간

grafana:  
  adminPassword: "myStrongPassword123\!" \# 초기 관리자 비밀번호 설정  
  persistence:  
    enabled: true  
    storageClassName: standard  
    size: 5Gi

#### 2.4 Helm 차트 설치

작성한 설정 파일을 적용하여 스택을 배포합니다. 릴리스 이름을 prometheus-stack으로 지정합니다.  
helm install prometheus-stack prometheus-community/kube-prometheus-stack \\  
  \--namespace monitoring \\  
  \-f my-values.yaml

#### 2.5 설치 확인

모든 구성 요소(Pod)가 Running 상태인지 확인합니다 5\.  
kubectl get pods \-n monitoring

### 3\. Grafana 접속 및 로그인

외부 접속 설정(Ingress 등)을 하지 않은 상태에서 로컬에서 접속하는 방법입니다 6\.

#### 3.1 포트 포워딩 (Port-Forwarding)

Grafana 서비스 포트(80)를 로컬의 3000번 포트로 연결합니다.  
\# Grafana 서비스 이름 확인 (보통 prometheus-stack-grafana)  
kubectl get svc \-n monitoring

\# 포트 포워딩 실행  
kubectl port-forward svc/prometheus-stack-grafana 3000:80 \-n monitoring

#### 3.2 로그인

1. 웹 브라우저에서 http://localhost:3000 접속  
2. **ID:** admin  
3. **PW:** values.yaml에 설정한 비밀번호 (설정 안 했다면 기본값 prom-operator 또는 Secret에서 확인 필요)

**비밀번호 Secret 확인 명령어:**  
kubectl get secret \--namespace monitoring prometheus-stack-grafana \-o jsonpath="{.data.admin-password}" | base64 \--decode ; echo

### 4\. Grafana \- Prometheus 데이터 소스 연결

kube-prometheus-stack으로 설치 시 **자동으로 연결**되어 있는 경우가 많습니다. 하지만 수동으로 연결하거나 설정을 검증해야 할 때, **가장 중요한 것은 URL 설정**입니다 8\.

#### 4.1 데이터 소스 설정 메뉴 진입

1. Grafana 왼쪽 사이드바에서 **Connections (또는 Configuration) \> Data Sources** 클릭  
2. Add data source 클릭 후 **Prometheus** 선택

#### 4.2 HTTP URL 설정 (핵심)

Grafana는 쿠버네티스 클러스터 **내부**에서 Prometheus와 통신해야 하므로, localhost가 아닌 **쿠버네티스 내부 DNS 주소**를 입력해야 합니다 10\.

* **URL 형식:** http://\<서비스이름\>.\<네임스페이스\>.svc.cluster.local:\<포트\>  
* **서비스 이름 확인:** kubectl get svc \-n monitoring 명령어로 Prometheus 서버의 서비스 이름 확인 (예: prometheus-stack-kube-prom-prometheus)

**입력 예시:**  
http://prometheus-stack-kube-prom-prometheus.monitoring.svc.cluster.local:9090

#### 4.3 저장 및 테스트

하단의 **Save & Test** 버튼을 클릭합니다.

* 성공 시: 녹색 박스로 Successfully queried the Prometheus API 메시지 출력 11\.  
* 실패 시: URL이 정확한지, Prometheus 파드가 정상 동작 중인지 확인해야 합니다.

### 5\. 대시보드 구성 (마무리)

연결 후에는 처음부터 대시보드를 만들지 않고, 커뮤니티 대시보드를 Import 하여 즉시 모니터링을 시작할 수 있습니다 12\.

* **Dashboards \> Import** 메뉴 클릭  
* **Import via grafana.com** 입력창에 ID 입력 후 Load 클릭  
* **Kubernetes Cluster (Prometheus):** 6417 또는 315  
* **Node Exporter Full:** 1860  
* 하단 **Prometheus** 드롭다운에서 위에서 연결한 데이터 소스를 선택하고 Import 클릭

이 과정을 통해 쿠버네티스 클러스터의 상태(CPU, 메모리, 네트워크 등)를 시각화하는 모니터링 환경 구축이 완료됩니다.
