# 노션 주간 회고 작성 스킬

## 개요
'일지 및 회고' 데이터베이스(Diary)에서 주간 일정 페이지를 찾아 해당 주에 기록된 WORK/LIFE 활동을 분석하고, KPT(Keep, Problem, Try) 템플릿으로 회고록을 작성하여 페이지 하단에 추가하는 스킬입니다.

## 사용 시나리오
```
사용자: "이번 주 회고 작성해줘"
→ 이번 주 일정 페이지 찾기
→ WORK/LIFE 섹션 내용 분석
→ KPT 템플릿으로 회고 작성
→ 페이지 하단에 회고 추가
```

## 2단계: 해당 주 일정 페이지 찾기

### 방법 1: 현재 주차 기준 검색 (기본)
현재 날짜를 기준으로 이번 주의 주차를 계산하여 검색합니다.

```
1. 오늘 날짜 확인 (예: 2026-01-25)
2. 이번 주 주차 계산 (예: week 04)
3. 검색 쿼리 생성 (예: "week 04 2026")
```

**Notion Search 도구 사용**:
```json
{
  "query": "week 04 2026",
  "query_type": "internal"
}
```

### 방법 2: 특정 주차 지정 검색
사용자가 특정 주차를 지정한 경우:

```
사용자: "지난 주 회고 작성해줘"
→ 현재 주차 - 1로 계산
→ 해당 주차 검색
```

**중요**: 
- 검색 결과에서 "일지" 페이지를 선택 (식단 페이지 제외)
- 페이지 URL/ID를 확인하고, 해당 페이지를 다시 fetch하여 전체 내용을 확인

## 4단계: KPT 회고 템플릿

### KPT 프레임워크 설명
- **Keep (유지)**: 잘 했고, 앞으로도 계속 유지할 것들
- **Problem (문제)**: 문제가 되었거나 개선이 필요한 것들
- **Try (시도)**: 다음에 시도해볼 것들, 개선 방안

### 회고 템플릿 구조
```markdown
## 📝 주간 회고

### Keep (유지할 것) ✅
- **항목 제목**: 구체적인 설명
- **항목 제목**: 구체적인 설명
- **항목 제목**: 구체적인 설명

### Problem (개선할 것) ⚠️
- **항목 제목**: 구체적인 설명
- **항목 제목**: 구체적인 설명

### Try (시도할 것) 🎯
- **항목 제목**: 구체적인 설명
- **항목 제목**: 구체적인 설명

### 한 줄 소감
> 이번 주를 한 마디로 정리하는 문장
```

## 5단계: 회고 작성 워크플로우

### 전체 프로세스
1. **주간 페이지 찾기**: Search로 해당 주차 "일지" 페이지 검색
2. **페이지 내용 조회**: Fetch로 페이지 전체 내용 확인
3. **일정 내용 분석**: Week List의 WORK/LIFE 완료/미완료 분석
4. **KPT 회고 작성**: 분석 내용 기반 회고 생성
5. **회고 추가**: 페이지 하단(`<empty-block/>` 다음)에 회고 섹션 추가
6. **결과 확인**: 추가된 내용 확인

### 회고 분석 가이드라인

#### Keep 작성 기준 (완료된 항목 `[x]` 기반)
- 계획대로 완료한 업무/일정
- 꾸준히 유지한 습관 (운동, 학습 등)
- 문제를 해결한 경험
- 생산성이 높았던 순간

#### Problem 작성 기준 (미완료 항목 `[ ]` 기반)
- 미완료된 계획이나 일정
- 동시에 너무 많은 업무 진행
- 예상과 다르게 진행된 업무
- 시간 관리 실패

#### Try 작성 기준 (Problem에서 도출)
- Problem에서 도출된 개선 방안
- 다음 주 우선순위 조정
- 습관 형성/개선 계획
- 구체적이고 실행 가능한 액션

## 9단계: 회고 품질 가이드라인

```
❌ "열심히 일했다"
✅ **스크립트 자동화**: 통합몰 미등록 업체 추출 스크립트를 작성하여 반복 작업 효율화
```

**2. 실행 가능한 Try**
```
❌ "더 잘하자"
✅ **OJT 집중 처리**: 네이버웍스/계정/개발환경 설정을 월~화 내 완료 목표
```

**3. 균형 잡힌 관점**
- WORK와 LIFE 모두 다루기
- 완료 항목과 미완료 항목 균형있게 분석
- 자기비판보다 성장 관점

**4. 적절한 분량**
- Keep: 3-5개 항목
- Problem: 2-4개 항목
- Try: 2-4개 항목
- 한 줄 소감: 1-2문장

### 분석 시 고려사항

**완료율 체크**:
- `[x]` 완료 항목 → Keep 후보
- `[ ]` 미완료 항목 → Problem 후보
- 하위 항목까지 확인하여 부분 완료 파악

**패턴 인식**:
- 주 초반 vs 후반 업무 분포
- 카테고리별 완료율 (협력사, Billing 등)
- LIFE 활동 실행 여부

## 베스트 프랙티스

### 1. 검색 시 "일지" 페이지 선택
- 검색 결과에 "식단" 페이지도 나올 수 있음
- 반드시 "일지"가 포함된 페이지 선택

### 2. 완료/미완료 구분 명확히
- `[x]`: 완료된 항목
- `[ ]`: 미완료 항목
- 하위 항목도 개별 체크

### 3. 삽입 위치 정확히
- `</columns>\n<empty-block/>` 다음에 삽입
- Copyright 이전에 위치

### 4. Bold 제목 형식 사용
- `**제목**: 설명` 형식으로 가독성 향상
- 각 항목이 명확하게 구분되도록

## 에러 처리

### 일반적인 에러와 해결방법

**에러**: "해당 주차 페이지를 찾을 수 없습니다"
- **원인**: 해당 주차의 일정 페이지가 없음
- **해결**: 사용자에게 알리고, 페이지 생성 여부 확인

**에러**: "Selection을 찾을 수 없습니다"
- **원인**: 페이지 구조가 예상과 다름
- **해결**: 페이지를 다시 fetch하여 실제 구조 확인 후 selection 수정

**에러**: "이미 회고가 존재합니다"
- **원인**: 동일 주차에 회고가 이미 작성됨
- **해결**: 사용자에게 덮어쓰기/추가/유지 선택 요청
